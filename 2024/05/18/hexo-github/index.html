將 Hexo 部署至 Github pages

發表於 2024-05-18 更新於 2024-05-20 分類於 hexo Disqus： 文章字數： 555 所需閱讀時間 ≈ 2 分鐘

前言

為什麼選擇使用 Github 的 pages，因為 hexo 有整合 Github pages 套件可以自動更新部署。

在 Github 開啟新的 Repositories

Repository name 的地方一定要填 [ 你的github帳號 ] .github.io ，我的帳號是 nilson-hsieh，所以我就要輸入 nilson-hsieh.github.io，然後按 [Create Repository] 。

之後網頁的網址：http://github帳號.github.io

安裝 hexo 套件

hexo 雖然有整合了Github page，但是預設是沒有安裝，所以需要執行下面指令去安裝

$npm install hexo-deployer-git --save

設定部署的資料

在設定文件 _config.yml 中修改網頁部署的方式

  • type: 部署的方式
  • repo: Repository 的資訊
  • branch: 要部署的分支

設定完成

完成設定後，在本機端編輯完成後執行下列指令就會自動更新到 Github pages

$ hexo deploy

使用自己的域名

Github pages 的網址預設是 http://github帳號.github.io ，如果有自己的網址名稱可以設定成自己的

設定指向

設定方式有分兩種

  • 使用頂級域名
  • 子域名

1.設定頂級域名

如果使用Ａ紀錄要將網址指向

IPv4 位址
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

如果使用 AAAA 紀錄

IPv6 位址
2606:50c0:8000::153
2606:50c0:8001::153
2606:50c0:8002::153
2606:50c0:8003::153

2.設定子域名

使用 CNAME 將子域名指向至 github帳號.github.io

設定 Custom domain

  1. 前往 hexo 所在的 Repository ，進入設定，右側 Pages 選單
  2. 在 Custom domain 欄位輸入要綁定的網址
  3. 按下確認後，等待網址檢查沒問題後就完成了

!! 這邊需要特別注意 !!

如果使用 CNAME Github 會自動產生一個名稱為 CNAME 的檔案，要將他複製到 hexo source 資料夾裡面，不然之後自動部署後會將這資料蓋掉，導致網址失效
